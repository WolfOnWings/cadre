## 2026-04-28 — task-mx simplified; brainstorm-cadre Double Diamond + review ensemble

### Narrative

The session covered three distinct arcs. The first: the user opened with a concern that handoff and task-mx workflows had grown too sub-heavy. Handoff was judged working well and left alone (startup latency noted but deferred). For task-mx, the entire subagent + trigger/surface hook infrastructure was collapsed into a lightweight orchestrator skill. The key insight driving the pivot: the machinery (score.ts, inbox, shard format) was already solid — the subagent was only doing what the orchestrator could do inline. The new task-mx-cadre skill is a thin glue layer: parse intent → draft shard with rich body → user review gate → write to inbox → call `score.ts --integrate`. Hooks and agent files retired. DEFERRED/SUPERSEDED status enum values also dropped (covered by weight manipulation and blocking). Smoke test passed E2E; PR #23 merged.

The second arc: agent-output path conventions were broken — files in per-dispatch wrapper subdirs, ISO timestamps in filenames. Fixed to a flat `<agent-name>/<slug>-MM-DD.md` convention; existing files renamed; researcher-cadre skill migrated to agent format (closing TODO #61 / ADR-061); PR #24 merged. The third and largest arc: plan-cadre was renamed to brainstorm-cadre and restructured around the Double Diamond (Discover → Define → Develop → Deliver-Direction). A canonical step-based planning format was defined this session and landed in CLAUDE.md; it also became the output format for brainstorm-cadre artifacts. Two new review primitives shipped alongside: brooks-review-cadre (skill) applies the Brooks rewrite-test reflection after Deliver-Direction; premortem-reviewer-cadre (agent) runs a structured failure-mode review. Staff-engineer-cadre dispatch timing was refined: after should-tree, not parallel. Brooks-review output format tightened to a three-line per-concern structure (Change / Justification / Outcome). A skill-template.md reference was drafted, user-edited, and landed in `.cadre/references/skill-template.md`; creator-cadre now points to it. All skills had Role Identity sections stripped (skills-don't-need-it rule). PR #25 merged.

User preferences sharpened throughout: they explicitly want plans rendered inline before persisting (not in a separate file to go find); depth-first decision-tree language should replace aside-killing language in CLAUDE.md and brainstorm-cadre anti-patterns; brooks-review concerns should be short parse-friendly structs, not paragraphs; future skill/agent creation should route through creator-cadre. One recurring signal: the orchestrator was drifting toward proposing additional ceremony (SOP label, PR ceremony on already-pushed main) and the user pulled it back to the simpler path each time.

### Decisions

- No new ADRs this session. ADR-061 (researcher-cadre skill → agent migration) closed via the agent-output PR.
- Doctrine update: canonical step-based planning format added to CLAUDE.md (Type / Mode / Scope header, ALL CAPS phases, `▸` bullets, `═══` double-rule, inline branching, `CLOSE` phase, INPUT/OUTPUT sentinels). This is now the shape for skills, agents, ad-hoc plans, brainstorm output.
- Doctrine update: depth-first decision-tree branching language replaced aside-killing in CLAUDE.md "Working with me" section and in brainstorm-cadre's anti-patterns.
- task-mx-cadre workflow: subagent retired; orchestrator skill is the canonical path. Score.ts is the sole board writer. DEFERRED/SUPERSEDED statuses removed from enum.
- Agent-output path convention: `.cadre/agent-output/<agent-name>/<slug>-MM-DD.md` (flat, no ISO, no wrapper subdir).
- Skills do not carry Role Identity sections (agent-only). Captured in skill-template.md.
