## Prose Summary

**Layer 1:** The direction's three custom checks (LOC, TODO grep, comment-ratio) are all specified as shell one-liners, but the load-bearing assumption "all 8 checks executable locally via `bun run check:<name>` with identical output" fails on Windows 11 without WSL — the dev machine for this project [1][2]. Moving LOC and TODO grep to Bun scripts (cross-platform, already the established Cadre idiom for non-trivial logic per ADR-050) closes the parity gap before any code is written. The comment-ratio check has a second problem independent of platform: no threshold or exit-code contract is defined in the direction, meaning the check cannot produce a binary verdict [3] — this must be decided before the script is authored. The `threshold=0%` coverage decision achieves the opposite of its stated forcing-function intent [4]: a zero threshold always passes even with no test files, so replacing it with a `check:test-exists` step (exit 1 if zero `.test.ts` files) is the correct mechanical forcing function. The aggregator's `needs:` list has a structural drift risk every time a new check job is added [5]: the direction's success criterion ("new tools pluggable as one-job-addition + one entry in aggregator needs") makes the aggregator the synchronization point — a comment-block discipline or an expression-based aggregator reduces that risk.

**Layer 2:** The Lipstyk-to-oxlint mapping is deferred to implementation without a spec [6] — producing the mapping table before writing the oxlint config prevents silent rule omissions and preserves the design decision record in a verifiable form.

**Layer 3:** The aggregator job label "All checks passed" will be semantically misleading on the first green PR when coverage is 0% and the custom checks are stubs [7] — renaming the aggregator to "CI complete (bootstrap)" gives the three-review architecture's merge-gate consumers a correct signal about what this baseline green means. The `package.json` scripts split between inline commands and `scripts/*.ts` files is unspecified, which will produce inconsistent implementation [8]: a simple ≤80-chars-inline / >80-chars-script rule applied at PR time costs five minutes and prevents `package.json` from accumulating multi-line shell pipelines in string values.

**Layer 4:** No UI or microcopy surface in a CI workflow spec — L4: no applicable surface.
