# Staff-Engineer Review — `task-mx-cadre`

**Target:** `task-mx-cadre` design (LLM-curated task board agent owning `.cadre/todos.md`).
**Calibration:** "Nothing is overkill. An LLM does this, not a human." Cost model is design+compute, not human cognitive load.
**Mode:** plan-mode artifact (orchestrator persists to `.cadre/agent-output/staff-engineer-cadre-output-task-mx-cadre/optimization-plan.md` post-ExitPlanMode).
**Precedent read:** `.claude/agents/handoff-mx-cadre.md`, `.claude/hooks/handoff-mx-{prime,logger,surface}-cadre.ts`, `.claude/agents/handoff-mx-cadre.parse.ts`, `.claude/settings.json`, `.cadre/todos.md`.

## Prose summary

The design is structurally sound and earns its mirroring of `handoff-mx-cadre`'s three-hook pattern, but the cost-profile divergence (mutation frequency ~10× handoff-mx's; handoff-mx synthesizes once per session boundary while task-mx fires on every TODO mutation) demands two adjustments at the harness layer: **debounce the trigger** [1] so an intake burst coalesces into one dispatch, and **split the trigger hook from the logger** [7] so the per-Write cost on the entire project doesn't pay for task-mx's classification logic on every event. The single largest leverage move is the **deterministic-script split** [2,4]: scoring polynomials, Kahn topological sort, critical-path DP, and Index markdown rendering belong in `task-mx-cadre.score.ts` (a direct sibling to `handoff-mx-cadre.parse.ts`), and the LLM's job collapses to (a) natural-language → front-matter extraction, (b) rich-expansion of sketch entries, (c) conflict-resolution on merge — exactly the boundary where LLM judgment dominates and exactly where the *Faith and Fate* anti-strengths (arithmetic, exact recall, true novelty) bite if you don't carve them out. The **single-writer / inbox-shard pattern** [3] resolves the atomic-write race by construction: orchestrator writes one shard per intake to `.cadre/todos.inbox/`, the agent is the sole writer of `todos.md`, and crash-recovery comes free via the rename-to-`.processing` discipline already proven in `handoff-mx-prime`. **Loop-guard becomes path-discrimination** [8] rather than a lock file — cleaner because there's no race between "agent finishes write" and "lock cleared," and the inbox naming convention already does the work. **Cycle detection refuses-and-emits** [9] to a `.rejected/` dead-letter directory: the user's prose is preserved, the DAG invariant on `todos.md` is preserved, and the failure is a durable artifact rather than a log line. A **Reflexion-scoped self-critique pass** [5] catches the four most likely bug classes (front-matter incomplete, orphan blocker refs, schema drift, duplicate-by-title) without entering a loop, mirroring `handoff-mx-cadre.refs.md`'s watchlist discipline. **Scoring weights live in `.cadre/config/task-mx-weights.json`** [6] so coefficients are data, not prompt — the user can A/B weight tweaks without touching the agent. The **schema is YAML front-matter** [12] so the deterministic script parses without LLM help and a future `query-todos.ts` becomes trivial; **inline-emoji shorthand is rejected** for non-greppability. The **I/O contract** [10] declares the three-namespace footprint (config / inbox / board) explicitly, and **tool grants stay at Read/Write/Bash** [11] mirroring handoff-mx exactly. At the interaction layer, **Index renders as three top-level sections with markdown tables** [13] for skim-time, with urgency-term breakdown progressively-disclosed in `<details>` blocks; **orchestrator-side intake is structured prose with YAML front-matter** [14] so dictation feels like dictation; **"pull next" is a deterministic read of the pre-rendered Index** [15] with zero subagent dispatch (Nielsen <500ms target); **dispatch surface is a one-liner** [16] from `{ok, reason}`, mirroring handoff-mx's pattern; **mutation discipline is bookended in the agent SOP** [17] at the attention peaks per *Lost in the Middle*.

## Per-layer confidence calibration

- **L1 (performance):** *high*. Debounce, deterministic script-split, and single-writer discipline are all standard patterns with predictable wins; the script-split has direct precedent in `handoff-mx-cadre.parse.ts`. Numeric targets are conservative.
- **L2 (LLM capability surface):** *high* on the cleavage [4] and config-as-data [6] (clear anti-pattern targets); *medium* on the self-critique watchlist [5] — the four rules are reasonable but the actual failure-mode distribution will only emerge in production use; revise after first 20 dispatches.
- **L3 (harness architecture):** *high* on hook split [7], path-discrimination loop-guard [8], inbox shards [3], and footprint discipline [10,11] — all mirror proven `handoff-mx` patterns or are direct least-privilege applications. *Medium* on cycle-detection rejection [9] — the dead-letter pattern is sound but the user-facing surface ("how does the rejection feel?") wants live calibration.
- **L4 (interaction design):** *medium-high*. Three-column table render [13], pull-next-as-read [15], and one-line surface [16] are well-grounded in Nielsen / Krug / Norman heuristics, but the actual feel (especially the orchestrator's intake-drafting prose [14]) wants a real-use rep before locking the microcopy. The bookended SOP rules [17] are load-bearing and worth installing immediately.

- **Cross-reference to ADR log / commit log / handoff-mx.** Surviving direction excludes; the cross-ref shape is itself a TODO #13 dependency.
- **Multi-board support** (per-project boards, per-branch boards). YAGNI.
- **Web/TUI surface for the board.** The markdown render IS the surface; a richer UI is shape-without-substance until a real workflow demands it.
